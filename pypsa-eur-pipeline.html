Open energy-system modelling · Iberia edition

Forty datasets,
thirty rules,
one continent.

PyPSA-Eur is the open, automated software pipeline that assembles a load-flow-ready optimisation model of the European energy system — electricity, heat, transport, industry, gas, hydrogen and CO2 — from raw, freely available data. This page walks the entire pipeline end-to-end and points at how it slices for Portugal and Spain.

Spatial scope ENTSO-E area
Default voltage ≥ 220 kV AC + HVDC
Grid topology OSM (prebuilt)
Clustered to 50–250 nodes

1 · Retrieve

Pull from primary sources or the Zenodo data bundle.

2 · Build

Clean, rasterise, intersect, allocate, simplify, cluster.

3 · Compose

Tie generators, demand and storage onto the clustered grid.

4 · Solve

Linear optimisation with PyPSA — capacities, flows, costs.

01 · Sources

The data behind every bus.

PyPSA-Eur is uncompromisingly open. Every grid line, every power-plant pin, every demand curve and every land-use mask is sourced from a published, citable dataset — primary where possible, with stable archives mirrored on data.pypsa.org. 02 · Transformations

From raw bytes
to a solvable network.

The workflow is orchestrated by snakemake: each box below is one or more rules. Inputs flow downward. Outputs are .nc, .geojson, .csv and .tiff files. The whole chain converges at prepare_network, which produces the solvable PyPSA network — a single NetCDF file that the optimiser consumes.

01

retrieve_* · downloads Pull the bundles.

Snakemake retrieve rules fetch every dataset from its primary URL or from the Zenodo / data.pypsa.org archive snapshot. The versioning registry (data/versions.csv) pins each download. Bundles too large for git ship as zipped archives.

retrieve_databundle retrieve_cutout retrieve_osm_data retrieve_eurostat_data retrieve_jrc_idees retrieve_gas_infrastructure_data retrieve_gem_* retrieve_powerplants retrieve_cost_data retrieve_hotmaps retrieve_country_runoff
02

build_cutout · atlite Cut a weather slab.

Using the atlite library, ERA5 reanalysis is downloaded for the model bounding box and merged with SARAH-3 satellite irradiation (which overrides ERA5's solar fields). The output is a multidimensional NetCDF "cutout" indexed by time, y, x — covering pressure, temperature, soil temperature, 100 m wind speed, runoff, direct/diffuse/TOA influx, albedo and roughness.

in·CDS API · EUMETSAT CM SAF out·cutouts/europe-2013-sarah3-era5.nc iberia·bbox ≈ (-10.0, 35.5, 4.5, 44.5) — ~700 × 1100 grid cells at 0.3°
03

build_shapes · build_osm_boundaries · build_bidding_zones Draw the regions.

GeoJSONs are emitted for country shapes (with point-of-view variants for disputed borders), exclusive economic zones, NUTS3 administrative units and OSM ADM1 polygons where NUTS is unavailable. Bidding-zone geometries are merged from Electricity Maps and entsoe-py, with manual TYNDP fixups (Crete split from GR, southern Norwegian zones merged).

in·eu_nuts2021 · eez · osm_boundaries · bidding_zones_* out·country_shapes.geojson, offshore_shapes.geojson, nuts3_shapes.geojson, bidding_zones.geojson
04

clean_osm_data · build_osm_network · base_network Build the grid topology.

Raw OSM features (lines, cables, substations) are cleaned: voltage / circuits / cables / wires / frequency columns parsed, semicolon-separated cells split into rows, capacities distributed proportionally to circuit splits, line endings attached to substations. The cleaned features become a PyPSA base network with HVAC ≥ 220 kV lines, HVDC links, transformers, and Voronoi cells assigning every point on the map to its nearest substation.

in·osm · country_shapes out·networks/base.nc — ≈ 3000 substations, 6600 HVAC lines, 70 HVDC links across Europe
05

build_natura · build_ship_raster Pre-render the exclusion masks.

Natura 2000 protected-area polygons are rasterised to the cutout grid (much faster than per-cell vector intersection). The World Bank ship-density raster is downsampled to the same grid. Both become inputs to the next-stage land-eligibility analysis.

build_natura build_ship_raster
06

build_transmission_projects · build_line_rating Layer planned grid & dynamic ratings.

TYNDP and other configured transmission projects are concatenated, deduplicated, and converted into new-line / new-link / adjust-line / adjust-link / new-bus CSVs. In parallel, dynamic line ratings (DLR) are computed: a heat-balance model uses ERA5 wind & temperature to set hourly s_max_pu per line — net of resistive heating, solar gain and convective cooling.

build_transmission_projects build_line_rating add_transmission_projects_and_dlr
07

simplify_network Collapse to one voltage.

The multi-voltage AC network is electrically equivalenced to a single 380 kV layer, transformers are absorbed, multi-hop HVDC routes are reduced to single links, and stub branches with no demand or generation are pruned. The output is a topologically minimal network that survives spatial clustering reliably.

in·base.nc out·networks/base_s.nc · busmap_base_s.csv · regions_onshore_base.geojson · regions_offshore_base.geojson
08

cluster_network K-means down to N zones.

The simplified network is clustered into a user-specified number of zones using k-means on substation positions weighted by an electrical metric. Buses, lines, transformers and regions all collapse onto the cluster representatives. Resolutions of 37, 128, 256 and 512 are standard test cases — whole-Europe runs typically pick something in 50–250.

in·base_s.nc · hac_features out·networks/base_s_{N}.nc · regions_onshore_base_s_{N}.geojson iberia·at N=128, ≈ 8 PT + ≈ 14 ES onshore nodes; at N=512, ≈ 30 + 55
09

determine_availability_matrix Where can you actually build?

For each clustered region and each technology (solar PV, onshore wind, offshore-AC wind, offshore-DC wind), a fine-grained land-eligibility analysis runs against CORINE / LUISA classes (urban, forest, water removed), Natura 2000, GEBCO bathymetry (offshore depth limit), the ship-density raster, and the EEZ. The output is an availability_matrix per technology — the per-grid-cell share of land that is actually buildable.

in·cutout · corine · luisa · natura · gebco · ship_raster · regions_* out·resources/availability_matrix_{clusters}_{technology}.nc
10

build_renewable_profiles Hourly capacity factors per node.

The availability matrix is multiplied with the local capacity factor (computed by atlite from the cutout wind / solar data) to produce a generator layout per region. Generators are binned into resource classes (best 20% / next 20% / etc.), and atlite is rerun to extract an hourly availability time-series and total installable capacity for each (bus, technology, resource class). The maximum installable potential p_nom_max is the sum across grid cells.

out·resources/profile_{tech}.nc · dims: (year, bus, bin, time) iberia·solar CF ≈ 0.20-0.24 (best in EU); onshore wind in NW ES / coastal PT ≈ 0.35-0.45; offshore floating (PT) up to 0.55
11

build_hydro_profile Reservoir inflow series.

Daily runoff from the cutout is integrated over each country's catchment, calibrated against EIA historic hydro generation and rescaled by the country_runoff dataset for years outside the EIA window. The output is hourly inflow (MW) per country to dam reservoirs and pumped-hydro.

out·resources/profile_hydro.nc — (countries, time) iberia·PT hydro capacity ≈ 7 GW reservoir + pumped; ES ≈ 17 GW — both highly seasonal, wet-winter dependent
12

build_powerplants Map conventional fleet to buses.

The powerplantmatching database (consolidating GPPDT, JRC, OPSD, GEO, BEYONDCOAL, IRENASTAT, CARMA) is filtered per the config (powerplants_filter), augmented with custom entries (custom_powerplants.csv), and each plant is snapped to its nearest clustered bus. Optional "everywhere" carriers add zero-capacity placeholder plants at every node for future expansion.

out·resources/powerplants_s_{clusters}.csv — name, fuel, tech, country, capacity, commissioning year, lat/lon, bus iberia·~80 ES conventional units (gas CC, nuclear: Almaraz · Ascó · Cofrentes · Trillo · Vandellós, coal closures); PT: Sines & Pego retired, Tapada & Lares CCGT
13

build_electricity_demand · build_electricity_demand_base · build_gdp_pop_non_nuts3 Demand as a time series.

Country-level OPSD / ENTSO-E hourly load is gap-filled (linearly for short gaps, period-copy for long gaps). Synthetic weather-correlated demand backfills any country-year missing. The country total is then disaggregated onto buses top-down using NUTS3 population × GDP per capita weights (with the JRC Energy Atlas as a finer 1 km raster fallback).

in·opsd_electricity_demand · entsoe_electricity_demand · synthetic · nuts3_population · gdp_per_capita out·load CSV at country level → bus-level load attached in add_electricity
14

build_monthly_prices Fuel & CO2 price track.

Monthly World Bank commodity prices and Instrat EU-ETS CO2 allowance prices are aligned to the modelled year and converted to €/MWhth and €/tCO2.

15

add_electricity Compose the electricity network.

The keystone rule. Onto the clustered network, add_electricity attaches:

  • today's transmission topology & transfer capacity (incl. under-construction lines if configured),
  • today's thermal & hydro generators with their capacities & commissioning years,
  • bus-level load time series,
  • extendable generators (initial capacity = 0) for PV, onshore-wind, offshore-AC-wind, offshore-DC-wind, OCGT, CCGT — each with hourly capacity factors from stage 10,
  • extendable storage: H2 & battery as StorageUnits, or as Store+Link+Link trios with independent charge / discharge / energy investment variables.
out·networks/base_s_{clusters}_elec.nc — a complete electricity PyPSA network
16

prepare_network Apply policy & reduce time.

Carbon budgets, carbon prices, N−1 security margins, transmission expansion cost/volume caps, and temporal resolution reduction (averaging or tsam segmentation) are applied here. The output is the file the solver actually reads.

out·networks/base_s_{clusters}_elec_{opts}.nc
17

sector coupling · prepare_sector_network Heat, transport, industry, gas, H2, CO2.

If sector coupling is enabled, additional layers wire in: residential / services heat (heat pumps with time-varying COP from cutout temperatures, gas/oil boilers, resistive, solar thermal, district heating), road / rail / shipping / aviation transport (EV fleets with mobility-profile constraints, H2 vehicles, ammonia / methanol / FT shipping), industrial process heat & feedstocks (DRI steel, methanol/ammonia/HVC chemistry from JRC-IDEES + Hotmaps + GEM steel/cement), the methane and hydrogen pipeline networks (SciGRID gas + Europe Gas Tracker, with retrofit options to H2), and CO2 capture / transport / storage (DAC, point sources, CO2StoP sites, optional pipeline build-out).

build_heat_demand build_cop_profiles build_transport_demand build_industrial_demand build_biomass_potentials build_gas_network build_gas_input_locations cluster_gas_network build_co2_sequestration_potentials build_salt_cavern_potentials prepare_sector_network
18

solve_network · solve_sector_network Optimise.

A linear (or mixed-integer) programme is handed to HiGHS / Gurobi / CPLEX. Objective: minimise total annualised system cost. Decision variables: line / link capacity expansion, generator capacity expansion, storage capacities, hourly dispatch of everything. Constraints: power balance per bus and hour, technical limits, transmission, CO2 caps, policy. Result: an investment + dispatch plan — capacities, flows, prices, emissions — written back into the same NetCDF schema.

out·results/{run}/networks/base_s_{clusters}_elec_{opts}_solved.nc
03 · Iberia · A worked example

When the model looks at Portugal & Spain...

Schematic — not to scale. Real OSM network has ~120 PT+ES substations at full resolution.

Geography

Country shapes from eu_nuts2021 (NUTS0 → NUTS3). PT decomposes into 25 NUTS3 regions, ES into 59. EEZ from Marine Regions sets the offshore wind footprint — vast for both, especially the Portuguese deep-water Atlantic.

Grid topology

The OSM grid captures REN (PT) and REE (ES) ≥ 220 kV backbones, the four PT-ES interconnections (Vila Fria-Cartelle, Lindoso-Cartelle, Pocinho-Aldeadávila, Alqueva-Brovales), the two AC Pyrenees crossings to France, and the Spain-Morocco HVDC across the Strait. Iberia is famously the "energy island" of Europe — cross-Pyrenees capacity sits around 2.8 GW.

Renewables

Cutout cells over Iberia produce some of the highest solar capacity factors in the EU (20-24% annual). Onshore wind in Galicia and the central Meseta hits 35-45%. Offshore wind is mostly floating territory (deep Atlantic shelf) and shows up in the DC-connected offshore tech.

Hydro

Inflow time series from ERA5 runoff, rescaled to EIA generation. PT operates ~7 GW reservoir+pumped, ES ~17 GW — both highly seasonal, with the wet 2010 vs dry 2017 difference visible in profile_hydro.nc.

Gas & H2

SciGRID gas + GEM Europe Gas Tracker provide Sines, Huelva, Bilbao, Barcelona, Cartagena, Mugardos LNG terminals plus the Iberian pipeline network. The single, low-capacity link to France keeps Iberia a near-island on the gas side too — making the H2Med / BarMar corridor an interesting "extendable link" in scenario runs.

Demand

Hourly load from OPSD & ENTSO-E (REE / REN feeds). Country totals are disaggregated by NUTS3 population × GDP per capita — concentrating load on Madrid, Barcelona, Valencia, Lisbon, Porto, with industrial pockets around Bilbao, Tarragona and Setúbal.

Industry

From Hotmaps and GEM trackers the model picks up ArcelorMittal in Asturias, cement plants across both countries, and large chemical sites in Tarragona, Huelva and Sines. These set fixed industrial heat & electricity demand, plus hydrogen and CCS targets.

Storage potentials

Salt caverns from Caglayan et al. show very little potential under Iberia (no Zechstein basin). The model therefore tends to favour battery + ammonia / methanol for long-duration storage in PT & ES scenarios.

04 · Output

What you end up with.

A single PyPSA NetCDF file holds the entire solved system. Buses, lines, links, generators, storage units, stores, loads — with hourly time-series for dispatch and prices, and per-component capacities. The result is queryable with the PyPSA Python API or any NetCDF tool.

results/iberia_run/ ├── networks/ │ ├── base.nc # raw OSM topology, EU-wide │ ├── base_s.nc # simplified to 380 kV equivalent │ ├── base_s_128.nc # clustered to 128 nodes │ ├── base_s_128_elec.nc # + generators, demand, storage │ └── base_s_128_elec_Co2L-3H_solved.nc # solved: CO₂ cap, 3-hour resolution ├── resources/ │ ├── profile_solar.nc # (year, bus, bin, time) │ ├── profile_onwind.nc │ ├── profile_offwind-ac.nc │ ├── profile_offwind-dc.nc │ ├── profile_hydro.nc │ ├── availability_matrix_128_solar.nc │ ├── powerplants_s_128.csv │ ├── regions_onshore_base_s_128.geojson │ └── regions_offshore_base_s_128.geojson ├── cutouts/ │ └── europe-2013-sarah3-era5.nc └── summaries/ ├── capacities.csv # MW per (country, carrier) ├── costs.csv # €/yr per (country, carrier, kind) ├── energy.csv # MWh per (country, carrier) └── prices.csv # €/MWh per (bus, time)

A typical Iberia-focused run with N=128, 3-hour temporal resolution, a 1990-baseline CO2 cap and sector coupling enabled solves in 15-60 minutes on a 32-core / 256 GB machine using HiGHS.
